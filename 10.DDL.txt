1. 스키마 정의
    1) DDL(Data Definition Language)
        -데이터베이스, 테이블을 CRUD 
        -테이블에 대한 정보는 메타데이터(Metadata)로 데이터사전(Data Dictionary)에 저장, 관리됨

    2) 데이터베이스(DB) 생성
        -CREATE DATABASE 데이터베이스명;

    3) 테이블 생성
        -CREAT TABLE 테이블명(컬럼명1 데이터타입(크기) (제약조건), 컬럼명2....);

2. 제약조건
    0) 테이블을 구성하는 각각의 컬럼들은 제약조건을 가질 수 있음
    1) NOT NULL
        -컬럼에 값에 NULL이 들어갈 수 없음
        -반드시 데이터 값이 들어가야 함
    2) UNIQUE
        -UNIQUE 컬럼에 들어가는 값은 테이블 내에서 유일함을 보장함
        -예: 직원 테이블에 직원번호는 해당 테이블 내에서 유일한 값이어야 함 
    3) Primay Key 
        -NOT NULL + UNIQUE 한 값이 들어가는 것을 보장함
        -예: 직원 테이블에 직원번호는 반드시 값이 있어야 하고 해당 테이블에서 유일한 값이어야 함
    4) CHECK
        -컬럼의 값에 사용자가 지정하는 특정 표현식을 만족하는 데이터가 들어감 
        -예: 학점 컬럼은 0.00~4.50 범위의 값만을 허용함
    5) Foreign Key
        -해당 컬럼의 값은 참조하는 부모 테이블에 PK로서 존재하는 것을 보장함

3. 스키마 수정
    1) 이미 생성된 스키마에 대해 수정할 경우 사용함
    2) 테이블 컬럼 추가/삭제/수정 
        -ALTER TABLE 테이블명 ADD 컬럼명 데이터타입 
        -ALTER TABLE 테이블명 DROP COLUMN 컬럼명
        -ALTER TABLE 테이블명 ALTER COLUMN 컬럼명 TYPE 데이터타입 
    3) 기본키 제약조건 추가/삭제
        -ALTER TABLE 테이블명 ADD PRIMARY KEY(컬럼명)
        -ALTER TABLE 테이블명 DROP PRIMARY KEY 
    4) UNIQUE 제약조건 추가/삭제
        -ALTER TABLE 테이블명 ADD UNIQUE(컬럼명)
        -ALTER TABLE 테이블명 ADD CONSTRAINT 제약명 UNIQUE(컬럼명1, 컬럼명2)
        -ALTER TABLE 테이블명 DROP UNIQUE 제약명
    5) 외래키 제약조건 추가/삭제
        -ALTER TABLE 테이블명 ADD FOREIGN KEY(컬럼명) REFERENCES 원테이블(원컬럼명)
        -ALTER TABLE 테이블명 DROP FOREIGN KEY 컬럼명
    6) 테이블명 
        -ALTER TABLE 테이블명 RENAME NEW_테이블명
    7) CHECK 제약조건 추가/삭제
        -ALTER TABLE 테이블명 ADD CHECK 조건
        -ALTER TABLE 테이블명 DROP CHECK 조건명
    8) DEFAULT 제약조건 추가/삭제
        -ALTER TABLE 테이블명 ALTER 컬럼명 SET DEFAULT 기본값
        -ALTER TABLE 테이블명 ALTER 컬럼명 DROP DEFAULT 

4. 스키마 삭제
    1) 데이터베이스(DB) 삭제
        -DROP DATABASE 데이터베이스명 
    2) 테이블 삭제
        -DROP TABLE 테이블명 
            -내용과 테이블 전체 삭제 
        -TRUNCATE TABLE 테이블명 
            -테이블 내용만 삭제
            -테이블 스키마는 유지
