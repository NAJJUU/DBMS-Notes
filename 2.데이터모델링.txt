1. 데이터 모델링
    -테이블(데이터)의 근간이 되는 것이 데이터 모델이며 데이터 모델을 설계하는 과정

2. 모델링 정의
    1) 우리가 살고 있는 3차원의 현실 세계를 단순화하여 표현하는 것
        -서로가 약속한 규약을 표기법이나 언어로 표현 
    2) 현실 세계를 추상화하여 그 구조를 표현한 것
        -일정한 형식에 맞게 표현
    3) 현실 세계에 존재하는 사물이나 사건에 관한 관점 및 연관된 주체(사람 혹은 그룹)를 위해 명확하게 하는 것 
        -애매모호함을 제거하여 쉽게 표현
    =>복잡한 현실 세계를 추상화, 단순화, 명확화하기 위해 일정한 표기법으로 모델을 표현하는 기법

3. 데이터 모델링 정의
    1) 현실 세계의 비지니스를 IT 시스템으로 구현하기 위해 데이터 관점으로 업무를 분석하는 기법
    2) 약속된 표기법으로 표현하는 과정
    3) IT 시스템의 근간이 되는 데이터베이스를 구축하기 위한 분석 및 설계의 과정 
    4) 대부분의 IT 시스템은 관계형 데이터베이스(RDBMS)를 기반으로 구축되어 있음
    5) 데이터 모델링을 통해 정의된 데이터 모델을 기반으로 물리적인 데이터베이스가 구축되고 SQL문을 활용하여 
       데이터가 CRUD(insert, select, update, delete) 됨

4. 데이터 모델링의 3단계
    현실세계 ---------------------> 개념적 구조 
              개념 데이터 모델링        |
                                       | 논리 데이터 모델링
                                      \|/
    물리구조(DB) <----------------- 논리적 구조
                  물리 데이터 모델링
    1) 개념적 데이터 모델링
        -구현하고 있는 대상에 대해 포괄적인 수준의 데이터 모델링 
    2) 논리적 데이터 모델링 
        -기본키, 속성, 관계, 외래키
    3) 물리적 데이터 모델링 
        -논리 데이터 모델을 기반으로 실제 물리 DB 구축 

5. 데이터 모델링의 3요소
    1) 어떤 것(Thing)
        -엔티티(Entity)                     -인스턴스(Instance)
        -예시: 사원                         -예시: 이순신 사원
    2) 어떤 것의 성격(Characteristic of a Thing)
        -속성(Attribute)                                -속성값(Attribute Value)
        -예시: 사원은 사원번호, 사원명을 가진다.           -예시: 사원번호 1번의 사원명은 이순신이다.
    3) 어떤 것과 어떤 것 사이의 연관(Association between Thing)
        -관계(Relationship)                             -페어링(Pairing)
        -예시: 사원(엔티티)은 부서(엔티티)에 속한다.       -예시: 이순신 사원은 개발팀에 속한다.
     
6. ERD(Entity Relationship Diagram = UML) 그리는 순서
    1) 비지니스에 필요한 엔티티를 그리고 속성을 정의함 
    2) 앞서 그린 엔티티를 적절하게 배치
    3) 각 엔티티 간의 관계를 설정
    4) 설정한 관계의 관계명을 기술
    5) 설정한 관계의 참여도 기술
    6) 설정한 관계의 필수 여부 기술
        -데이터 모델링 표기법
            -Case Method(Barker 표기법)

7. dvdrental 데이터 모델 분석
    1) 하나의 카테고리(엔터티)는 여러 개의 필름카테고리(엔터티)를 가질 수 있다.
    2) 하나의 필름(엔터티)은 여러 개의 필름카테고리(엔터티)를 가질 수 있다.
    3) 하나의 언어(엔터티)는 여러 개의 필름(엔터티)을 가질 수 있다.
    4) 하나의 필름(엔터티)은 여러 개의 재고(=inventory, 엔터티)를 가질 수 있다.
    5) 하나의 필름(엔터티)은 여러 명의 필름액터(엔터티)를 가질 수 있다.
    6) 한 명의 액터(엔터티)는 여러개의 필름액터(엔터티)를 가질 수 있다.
    7) 하나의 재고(엔터티)는 여러 번의 걸쳐서 렌탈(=하나의 비지니스임, 엔터티)을 할 수 있다.
    8) 한번의 렌탈(엔터티)을 통해서 여러 번의 지불(엔터티)을 할 수 있다.
    9) 한 명의 스탭(엔터티)은 여러 번의 렌탈(엔터티)업무를 처리할 수 있다.
    10) 한 명의 스텝(엔터티)은 여러 번의 지불(엔터티)업무를 처리할 수 있다.
    11) 한 명의 고객(엔터티)은 여러 번에 걸쳐서 렌탈(엔터티)을 할 수 있다.
    12) 한 명의 고객(엔터티)은 여러 번에 걸쳐서 지불(엔터티)을 할 수 있다.
    13) 하나의 주소(=동네의 체인점 느낌, 엔터티)는 여러 명의 고객(엔터티)을 가질 수 있다.
    14) 하나의 주소(엔터티)는 여러 개의 상점(엔터티)을 가질 수 있다.
    15) 하나의 주소(엔터티)는 여러 명의 스텝(엔터티)을 가질 수 있다.
    16) 한 명의 스텝(엔터티)은 여러 개의 상점(엔터티)에 속할 수 있다.
    17) 하나의 국가(=country, 엔터티)는 여러 개의 도시(엔터티)를 가질 수 있다.
    18) 하나의 도시(엔터티)는 여러 개의 주소(엔터티)를 가질 수 있다.
